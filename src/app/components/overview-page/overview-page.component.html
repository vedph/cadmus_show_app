Overview

General Concepts

Here you can find a directory of Cadmus-related learning materials, together with the list of all its GitHub repositories and real-world projects using it.

Read and play

This site has been designed to be simple and to the point, leaving out most of technical and methodological discussions. It also provides a lot of interactivity in its pages, where users can play with the concepts illustrated, and even prepare some digital resources for using the system. In fact, technically this is a web application, rather than just a bunch of hypertexts.

Contents

The documentation found in these pages focuses on the most relevant Cadmus topics. You can find a more detailed technical documentation in the GitHub repositories. The main topics covered here are:

  • data architecture: an overview of the data architecture, introducing key terms in Cadmus lingo like items and parts, in the context of dynamic modelling built by composition.
  • text architecture: more focus on how text and metatextual data are handled in Cadmus. In fact, a radical departure from traditional markup here is that text has any number of separate, independent layers on top of it, including any type of metadata, each with its model; and that text is just a datum like any other, sharing the same architecture of the whole system.
  • taxonomies: a short discussion about taxonomies and shared vocabularies, freely defined and available across all the editors.
  • infrastructure: as Cadmus relies on databases rather than on file system, it can provide a robust, full-fledged editing environment, with all the features typically expected from a content creation system. Here you can find more about the general software architecture, and infrastructural features like real-time indexing and search.
  • profile: your data architecture, and the corresponding editing UIs, are configured in an external JSON document, including all the data required by the system to fit your own project. Here I introduce the main sections of the profile, which you can directly create and edit in this site using the profile builder.
  • models shop: finally, you can get a general feeling of the Cadmus editing experience by exploring the data models and their user interfaces in this sort of models shop. In fact, that's not only a metaphor, as Cadmus projects are built by composing data models and their editors, focusing on sharing and reuse.

GitHub Repositories

All the Cadmus repositories are public and hosted at the VeDPH GitHub. They are usually organized per-project: a set of general repositories provide the full infrastructure, and then each project has its own set of repositories.

Repos and projects

The number of repositories per project depends on the projects requirements; usually, a full-fledged Cadmus project can have a documentation repository, a models repository, an API repository, and a frontend web app repository.

All these repositories represent a very limited portion of the code, as most functionality is directly borrowed from the framework. In fact, some projects may even provide just a wrapper for the backend API and the frontend app (with their own homepage being the only resource typically built ad-hoc), as they mostly reuse components and even data models.

Docker Images

Running and deployment

All the images for the various Cadmus-based projects are public and hosted in the VeDPH Docker Hub.

The easiest way to play with a Cadmus project or setup it on a server is just downloading a Docker-compose script and run it, whatever your platform (Linux, Windows, or Mac). No dependencies are needed except for Docker itself.

Encoding a critical apparatus

Daniele Fusi and Daniel Kiss, Encoding a critical apparatus. Webinar held ON LINE in December 7, 2020 within the Webinar series in Theories and Practices of the Annotation through Domain-Specific Languages.

This seminar focuses on the challenge of remodelling legacy digital content to create highly structured digital critical editions, by using my Proteus conversion system coupled with Cadmus as the target content editor. The real-world usage case is represented here by Daniel Kiss Catullus Online, where the apparatus is being remodelled also in the perspective of major integration with resources like Musisque Deoque. A deeper insight in critical apparatus can be found in the above video.
